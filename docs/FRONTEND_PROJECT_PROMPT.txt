## PROJECT OVERVIEW

Build a **modern, secure file-sharing web application** that hides sensitive 
files inside innocent-looking media (images, audio, video) with military-grade 
encryption and multi-user support.

Users can hide ANY file in ANY media type using LSB steganography, encrypted 
with RSA-2048 + AES-256, with support for multi-recipient secure sharing.

---

## KEY FEATURES

### 1. NINE STEGANOGRAPHY CONCEPTS
Users select from 9 cross-media hiding methods:

    🖼️  Image → Image        (hide image in image)
    🎬  Image → Video        (hide image in video)
    🔊  Image → Audio        (hide image in audio)
    ⚡  Video → Video        (hide video in video) [GPU-accelerated]
    📹  Video → Image        (hide video in image)
    🎥  Video → Audio        (hide video in audio)
    🎵  Audio → Audio        (hide audio in audio)
    🎶  Audio → Image        (hide audio in image)
    🎼  Audio → Video        (hide audio in video)


### 2. AUTHENTICATION & USER MANAGEMENT
    ✓ User signup/login with JWT tokens
    ✓ Personal RSA key pair generation (2048-bit)
    ✓ Public key sharing for file encryption
    ✓ Private key management (encrypted storage)
    ✓ User profile with key management


### 3. FILE ENCRYPTION OPTIONS
Three encryption methods available:

    🔐 RSA-Only       - Encrypt with user's public key only
                       Highly secure, no password needed
    
    🔑 Password-Only  - AES-256 encryption with password only
                        Anyone with password can extract
    
    ⭐ Hybrid         - RSA + Password encryption (recommended)
                       Maximum security, password required


### 4. MULTI-RECIPIENT SHARING
    ✓ Hide file with User A's encryption
    ✓ Select Recipients B, C, D during hiding
    ✓ Only authorized recipients can extract
    ✓ Creator can always extract their own files
    ✓ Each recipient has encrypted DEK (Data Encryption Key)
    ✓ Metadata tracking (creator, recipients, timestamp)


### 5. PERFORMANCE FEATURES
    ⚡  4-6x speedup over standard steganography
    🎮 GPU acceleration (NVIDIA CUDA)
    📦 1GB+ file support (streaming mode)
    📊 Real-time progress tracking
    ⏱️  Expected: 30-45 seconds for 1GB hide


### 6. ADDITIONAL FEATURES
    ✓ Drag-and-drop file uploads
    ✓ Real-time file preview
    ✓ Download/share stego files
    ✓ Extraction with password validation
    ✓ File integrity verification
    ✓ History of uploads/downloads

## USER WORKFLOWS

### WORKFLOW 1: Individual File Hiding
    
    1. User signs up → RSA keys auto-generated
    2. Selects Concept (e.g., Image → Image)
    3. Uploads secret file + cover media
    4. Chooses encryption (Hybrid recommended)
    5. Sets password
    6. Clicks "Hide File" → See progress
    7. Downloads stego file
    8. Shares via email/cloud
    9. Recipient uploads stego file
    10. Recipient enters password
    11. Clicks "Extract" → Gets original file


### WORKFLOW 2: Multi-Recipient Sharing
    
    1. User A uploads file
    2. Selects Recipients: User B, User C, User D
    3. Chooses encryption (Hybrid)
    4. Sets password
    5. File encrypted once, 3 DEKs created (one per recipient + creator)
    6. User A downloads stego file
    7. User A sends to Users B, C, D (via email, etc.)
    8. User B extracts with their private key
    9. System asks for password
    10. File decrypted and extracted
    11. Users C and D can do the same


### WORKFLOW 3: Video Hiding (GPU-Accelerated)
    
    1. User selects Concept: Video → Video
    2. Uploads 1GB video (secret) + 4K video (cover)
    3. GPU automatically selected (detected)
    4. System processes in 32-frame batches
    5. Real-time progress: 50% → 75% → 100%
    6. Expected time: 30-45 seconds (vs 3-5 min on CPU)
    7. Downloads stego video uncompressed


### WORKFLOW 4: Message Hiding
    
    1. Selects "Hide Message" from menu
    2. Types secret message
    3. Selects concept (e.g., Audio → Audio)
    4. Uploads cover audio file
    5. Selects recipients (optional)
    6. Clicks "Hide Message"
    7. Downloads stego audio
    8. Recipient extraction same as file
